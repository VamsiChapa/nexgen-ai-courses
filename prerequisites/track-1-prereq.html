Day P1

What is a Computer & How Does Software Work?

Before writing a single line of code, you need a mental model of what a computer actually is, what software is, and how the programs you use every day actually come to life.

Core Concepts

Hardware vs Software

A computer has two sides: hardware (the physical parts you can touch) and software (the instructions that tell hardware what to do). Neither is useful without the other.

Think of hardware as a piano and software as the sheet music. The piano does nothing without music; music does nothing without a piano.

The Brain / Memory / Filing Cabinet Analogy

  • CPU (Central Processing Unit) — the brain. It executes instructions, one at a time, extremely fast. Modern CPUs execute billions of operations per second.
  • RAM (Random Access Memory) — short-term memory. It holds data the CPU is actively working on. When you shut down, RAM is wiped. This is why unsaved work disappears.
  • Storage (HDD/SSD) — the filing cabinet. It holds data permanently — your photos, documents, and installed programs. Much slower than RAM, but keeps data when powered off.

Why this matters for programming: when your code reads a file, it moves data from storage → RAM. When your program runs, the CPU reads instructions from RAM. This pipeline is why some programs load slowly — they're moving a lot of data.

Operating Systems

The operating system (Windows, macOS, Linux) is software that manages everything else. It handles:

  • Memory allocation — deciding which program gets how much RAM
  • File management — organising your storage into folders and files
  • Process management — running multiple programs at once
  • Hardware communication — acting as a translator between your code and physical hardware

When you write Python, you don't talk to the CPU directly. You talk to the OS, which talks to the CPU. This abstraction is why the same Python code runs on Windows, Mac, and Linux.

From Source Code to Running Software

You write code as plain text. The computer can't run plain text — it only understands machine code (binary: 0s and 1s). There are two main approaches to get from one to the other:

  • Compilation — a program called a compiler reads your entire source code and translates it into machine code upfront. The result is an executable file. Languages: C, C++, Go, Rust. Fast at runtime, but you must compile before running.
  • Interpretation — an interpreter reads and executes your code line by line at runtime. Languages: Python, JavaScript, Ruby. Slower, but easier to develop and test quickly.

Python is interpreted. When you run python hello.py, the Python interpreter reads your file and executes it. This is why Python is beginner-friendly — you write code, run it, see results immediately.

Day P2

The Internet & Digital Tools You Need

Modern programming is inseparable from the internet. Today you set up the digital tools you'll use throughout this course, including your coding environment and your very first developer account.

Core Concepts

How the Internet Works (Simply)

The internet is a massive network of computers that can send data to each other. When you open YouTube, your browser sends a request to YouTube's servers (computers in a data centre), and they send back the video data.

  • Client — your device (the requester)
  • Server — the computer holding the data (the responder)
  • HTTP/HTTPS — the language they use to communicate
  • URL — the address of a resource on a server (like a postal address)

HTTPS = secure HTTP. The "S" means the connection is encrypted. You should only enter passwords or personal info on HTTPS sites (the padlock icon in your browser).

Google Colab — Your Primary Coding Environment

Google Colab is a free, online Python environment that runs in your browser. No installation needed. It gives you:

  • A notebook interface — code and text mixed together
  • Free compute (including free GPU access for AI work)
  • Automatic saving to Google Drive
  • Easy sharing — like a Google Doc, but for code

Access it at colab.research.google.com. You'll need a Google account.

GitHub — Your Coding Portfolio

GitHub is where developers store and share their code. Think of it as Google Drive for code, but with superpowers. Every serious developer has a GitHub profile — it becomes your portfolio when applying for internships or jobs.

Go to github.com and create a free account. Choose your username carefully — this is your developer identity. Keep it professional (your name or a variation).

Day P3

Setting Up Your Learning Environment

You have two good options for writing and running Python. You'll learn both today, and understand when to use which one.

Path 1: Google Colab (Recommended to Start)

Colab is the easiest starting point. It requires no installation and works on any computer, Chromebook, or tablet. All your work saves automatically. The only thing you need is a Google account and internet access.

For this course, Colab is perfectly capable of handling everything through Week 6. You can switch to a local setup later when you're comfortable.

Path 2: Local Setup (Optional but Valuable)

Step 1 — Install Python

Go to python.org/downloads. Download Python 3.11 or newer. During installation, check the box "Add Python to PATH" — this is critical. Without it, your terminal won't recognise the python command.

Step 2 — Install VS Code

Go to code.visualstudio.com. Download and install VS Code — it's free. After installation, open VS Code and install the Python extension by Microsoft (search for it in the Extensions sidebar on the left).

Step 3 — Terminal Basics

The terminal (Command Prompt or PowerShell on Windows, Terminal on Mac/Linux) lets you control your computer with typed commands. Essential commands:

  • cd folder-name — change directory (navigate into a folder)
  • cd .. — go up one level
  • ls (Mac/Linux) or dir (Windows) — list files in current folder
  • python filename.py — run a Python file
  • python --version — check your Python version

Day P4

Math & Logic for Programming

Programming uses math — but not in the way your maths class does. The concepts are simpler than you think, and once you see them as code, they actually become more intuitive.

Core Concepts

Variables: Math vs Programming

In algebra, a variable like x is an unknown you solve for. In programming, a variable is a named storage location. You set its value; it doesn't need to be solved for.

Python
# In maths: x = 5 means "x is unknown, find it" # In Python: x = 5 means "store the value 5 in a box called x" x = 5 y = 10 total = x + y print(total) # Output: 15 # You can change variables — unlike in maths x = 20 print(x + y) # Output: 30

Boolean Logic: The Foundation of Decisions

Boolean logic deals with True and False. Every decision your code makes boils down to a Boolean. The three operators are:

  • AND — both conditions must be True
  • OR — at least one condition must be True
  • NOT — reverses the result
Python
is_raining = True have_umbrella = False # AND: need BOTH to be true stay_dry = is_raining and have_umbrella print(stay_dry) # False — raining but no umbrella # OR: need at least ONE to be true should_worry = is_raining or not have_umbrella print(should_worry) # True

Why "Math Isn't Scary" in Code

Here's the secret: when you write AI code, you don't do the maths yourself. You write code that describes the maths, and the computer does the calculation. Your job is to understand what calculation to use, not to compute it by hand.

Focus on what the maths means, not how to compute it manually. That intuition is what separates a good programmer from someone who just memorised syntax.

Day P5

Typing Speed & Keyboard Shortcuts

Typing code is different from texting. Precision matters more than speed at first, but both matter in the long run. Today you'll also learn the keyboard shortcuts that make programmers so efficient.

Why This Matters

Professional developers type code for hours every day. A programmer at 30 words per minute (WPM) loses roughly 40% of their coding time to typing friction compared to one at 60 WPM. More importantly, slowness breaks your flow state — that deep focus where you're solving problems, not hunting for keys.

Programming also uses characters that aren't in normal writing: { } [ ] ( ) _ = == != < > : ; # @. You need to find these without looking down.

Essential Shortcuts

Universal (All Applications)

  • Ctrl/Cmd + C — Copy selected text
  • Ctrl/Cmd + V — Paste
  • Ctrl/Cmd + Z — Undo (your best friend)
  • Ctrl/Cmd + Shift + Z — Redo
  • Ctrl/Cmd + A — Select all
  • Ctrl/Cmd + F — Find (search in document)
  • Ctrl/Cmd + S — Save (even in Colab, saves to Drive)

VS Code / Code Editor Shortcuts

  • Ctrl/Cmd + / — Toggle comment on current line
  • Alt + Up/Down Arrow — Move current line up or down
  • Shift + Alt + Down — Duplicate current line below
  • Ctrl + D — Select next occurrence of selected word
  • Ctrl + Shift + P — Command palette (search all actions)
  • Ctrl + ` — Open integrated terminal

Google Colab Shortcuts

  • Shift + Enter — Run current cell, move to next
  • Ctrl + Enter — Run current cell, stay in cell
  • Ctrl + M + B — Insert code cell below
  • Ctrl + M + M — Convert cell to Markdown (text)
  • Ctrl + M + Y — Convert cell to code

Day P6

Problem Solving Mindset

The most important skill in programming isn't knowing Python syntax. It's knowing how to think through problems. This is called computational thinking, and it's something you can practice deliberately.

Computational Thinking

The Four Pillars

  • Decomposition — break a complex problem into smaller, manageable pieces. "Build an AI chatbot" becomes: handle input → process text → generate response → display output. Each piece is solvable.
  • Pattern Recognition — notice similarities between problems you've already solved. Most programming problems have been solved before in some form. Recognising patterns helps you reuse solutions.
  • Abstraction — focus on essential information and ignore irrelevant details. When you call print(), you don't think about how Python renders text to the screen — you just use the abstraction.
  • Algorithm Design — create a step-by-step plan before writing code. An algorithm is just a precise recipe. Write it in plain English first; then translate it to code.

The Debugging Mindset

When your code doesn't work, the wrong question is: "Why is Python being stupid?" The right question is: "What assumption did I make that is wrong?"

Every error message tells you something. Python errors include:

  • SyntaxError — you typed something Python doesn't understand (usually a typo or missing colon/bracket)
  • NameError — you used a variable that doesn't exist yet
  • TypeError — you used the wrong type (e.g., added a number to a string)
  • IndexError — you tried to access a list item that doesn't exist

How to Google an error: copy the error type + message (e.g., "Python TypeError unsupported operand type") and paste it into Google. Add "stack overflow" to get direct answers. This is how every professional developer works.

Imposter Syndrome is Normal

Almost every programmer — from beginners to senior engineers — feels like they don't know enough. You will feel confused, frustrated, and stuck. This doesn't mean you're bad at it. It means you're learning.

The only difference between someone who succeeds and someone who doesn't is persistence. When something doesn't work, try one more thing. When you're confused, ask one more question. That's it.

Your First Real Python Program

Today everything comes together. You'll write a complete, multi-feature Python program from scratch — no copy-pasting. Every line is explained. By the end, you'll know you're ready.

The Program

You'll build a "Student Profile" program that: asks your name, age, and favourite subject; calculates your birth year; tells you what grade you'll be in 5 years; stores your info in a dictionary; and prints a formatted summary. Here's the full code — type it out (don't copy-paste).

Python — Student Profile Program
# ── Student Profile Program ────────────────────────────── # We ask the user questions and build a profile # input() pauses the program and waits for the user to type name = input("What is your name? ") age = int(input("How old are you? ")) # int() converts text to number subject = input("What is your favourite subject? ") grade = int(input("What grade are you in? ")) # ── Calculations ────────────────────────────────────────── # The current year minus age gives approximate birth year import datetime current_year = datetime.datetime.now().year birth_year = current_year - age future_grade = grade + 5 # ── Store in a dictionary ───────────────────────────────── # A dictionary stores key-value pairs, like a real dictionary profile = { "name": name, "age": age, "subject": subject, "grade": grade, "birth_year": birth_year, "future_grade": future_grade } # ── Print formatted summary ─────────────────────────────── # f-strings let you embed variables inside strings using {} print("\n─────────────────────────────") print(f" Student Profile: {profile['name']}") print("─────────────────────────────") print(f" Age: {profile['age']}") print(f" Born around: {profile['birth_year']}") print(f" Favourite: {profile['subject']}") print(f" Current grade: {profile['grade']}") print(f" Grade in 5 yrs: Grade {profile['future_grade']}") print("─────────────────────────────")
